Entendiendo el Posicionamiento CSS

Un recorrido por `static`, `relative`, `absolute`, `fixed` y `sticky`.

Contacto Rápido

1. Posicionamiento `static` (Por Defecto)

Este párrafo tiene `position: static`, que es el valor predeterminado. Los elementos estáticos se posicionan según el flujo normal del documento. Las propiedades `top`, `bottom`, `left`, `right` y `z-index` no tienen ningún efecto aquí.

Soy una caja estática.

El siguiente texto fluye normalmente después de la caja estática. Puedes ver que la caja ocupa su espacio en el documento.

2. Posicionamiento `relative`

Este es un texto normal antes de la caja con `position: relative`.

Soy una caja **relativa**.

Me moví 20px a la derecha y 20px abajo de mi posición normal.

A pesar de que la caja interna `relative-shifted-box` se movió, la `relative-box` original **sigue ocupando su espacio** en el flujo normal, como puedes ver con este texto que la rodea.

Este texto viene después del contenedor relativo. Observa que el espacio de `relative-box` sigue ocupado.

3. Posicionamiento `absolute`

Un elemento con `position: absolute` se saca del flujo normal del documento. Esto significa que otros elementos se comportarán como si no existiera.

Soy el padre posicionado (`relative`)

Mi hijo `absolute` se posicionará relativo a mí.

Soy una caja **absoluta**.

Estoy a 10px del borde derecho y 10px del borde inferior de mi padre posicionado.

El texto en el padre fluye como si la caja absoluta no estuviera. El espacio que "ocuparía" la caja absoluta es ignorado por los demás elementos. Esto es crucial para entender la diferencia con `relative`.

Más contenido para mostrar el flujo...

Este párrafo está fuera del padre. Se posiciona después del contenedor padre.

4. Posicionamiento `fixed`

Este elemento con `position: fixed` está siempre visible en la pantalla, sin importar cuánto hagas scroll. Es útil para barras de navegación o botones de acción flotantes.

¡Soy fijo!

Siempre en la misma posición.

El texto de la sección fluye normalmente, como si la caja fija no existiera. La caja fija se posiciona con respecto al viewport (la ventana del navegador).

5. Posicionamiento `sticky`

Este encabezado se pegará al tope

Este encabezado `sticky` se comportará como un elemento normal (`static` o `relative`) hasta que su posición de desplazamiento cruce el umbral (`top: 0` en este caso). Luego se "pegará" a esa posición como si fuera `fixed`.

¡Sigue haciendo scroll hacia abajo para ver el efecto! El encabezado se mantendrá visible mientras su contenedor padre esté en la vista.
